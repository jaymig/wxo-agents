User Context agent

This tutoral introduces of context variables and walks you through how to deploy an agent with the 'wxo_user_name' and 'wxo_email_id' context variables and demonstrate how a tool can read and display the values. This tutorial assumes that you have completed the ADK Hello World ADK tutorial. These two context variables are the best to start with, since the default watsonx Orchstate chat portal sets the value of the variables during the session. Custom context variables will be covered in a subsequent article.

More information on context varibles is located in this ADK documentation

In this tutoral you will: 1. Create a new agent that is configured with the 'wxo_user_name' and 'wxo_email_id' context varibles. 2. Add a tool that is able to read and display the context variables. 3. Interact with the agent and have it demonstrate the value of the user context variables.

Import the tools

  1. Download the example python tools and the reqiurements.txt file into your 'demo' folder.

  2. Import the tool file using the below command line 

    orchestrate tools import -k python -f ./user_context.py -r ./requirements.txt
    NOTE: It is important that you are running the 'orchestrate' command from the same directory as the user_context.py and requirements.txt files.

NOTE: This python file actually includes two tools.
You should see an output as follows: 

[INFO] - Successfully imported tool '...'

3. Confirm that the tools were installed by listing the existing tools 
orchestrate tools list

If you see a knowledge base with the name 'knowledge_base_for_agent_Hello_world', congratulations. It is time to import the agent that will use these tools.

Import the agent

The following steps guild you through importing the agent that is configured to use the tools that you just imported. If the tools were not imported, this step will fail with an error message.

  1. Download the agent spec file{target="_blank"} into your 'demo' folder.

  2. Import the agent configuration file with the following command 

    orchestrate agents import -f ./welcome_agent.yaml

    If successful, you should see the following message: 
    [INFO] - Agent 'welcome_agent' imported successfully

Congratulations! Your agent is imported. You can verify that it works by logging into Watsonx Orchestrate and chatting with it.
